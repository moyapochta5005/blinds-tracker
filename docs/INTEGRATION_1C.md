# Техническое задание: интеграция 1С с трекером заказов

## 1. Общая информация

**Заказчик:** ООО «Штора» (Махачкала, бренд tkani-05.ru)
**Контактное лицо:** Максим (основатель)
**Что интегрируем:** учётную систему 1С Предприятие (локальная, у заказчика на ПК) с веб-сервисом отслеживания заказов **Blinds Tracker**.

**Цель:** при оформлении заказа в 1С — автоматически создавать карточку заказа в трекере, чтобы покупатель получил ссылку/QR для отслеживания статуса.

## 3. Бизнес-логика

### 3.1. Сущности

- **Заказ** — товарный документ в 1С (накладная / заказ покупателя / реализация).
- **Покупатель** — физическое лицо, для которого изготавливают товар.
- **Установщик (дилер)** — сотрудник или партнёр, который оформил заказ от лица покупателя. У установщика есть закреплённый менеджер. Установщики уже занесены в трекер вручную или импортом.
- **Менеджер** — сотрудник компании. В трекер не передаётся напрямую — определяется автоматически по установщику.

### 3.2. Связь установщик → менеджер

В трекере у каждого установщика проставлено поле manager_id. Когда 1С присылает заказ с указанием установщика, трекер автоматически назначает менеджера = менеджера этого установщика.

### 3.3. Идентификация установщика — два способа

| Способ | Поле в запросе | Когда использовать |
|--------|----------------|--------------------|
| По ID | installer_id (число) | Если в 1С хранится идентификатор установщика из трекера (рекомендуется) |
| По телефону | installer_phone (строка +7XXXXXXXXXX) | Если в 1С хранится только телефон установщика |

Приоритет: если переданы оба поля — трекер использует installer_id.

Если установщик не найден — заказ всё равно создаётся, но без привязки к менеджеру (менеджер назначит его вручную через панель сотрудника).

## 4. API эндпоинты

Все запросы должны содержать заголовки:

```
Content-Type: application/json
X-API-Key: tkani05-secret-key-2026
```

### 4.1. Создание заказа

`POST https://track.tkani-05.ru/api/v1/integration/orders`

**Тело запроса (JSON):**

```json
{
  "external_id": "ZK-2026-001234",
  "customer_name": "Иванов Иван Иванович",
  "customer_phone": "+79991234567",
  "product_name": "Рулонные шторы Blackout 120x180, цвет белый",
  "installer_id": 1,
  "installer_phone": "+79876543210",
  "comment": "Доставка после 18:00"
}
```

**Описание полей:**

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| external_id | string (1..100) | Да | Уникальный номер заказа из 1С |
| customer_name | string | Да | ФИО покупателя |
| customer_phone | string | Да | Телефон в формате +7XXXXXXXXXX (11 цифр после +) |
| product_name | string | Да | Название товара |
| installer_id | integer | нет | ID установщика в трекере |
| installer_phone | string | нет | Телефон установщика в формате +7XXXXXXXXXX |
| comment | string | нет | Дополнительный комментарий |

**Успешный ответ (HTTP 201 Created):**

```json
{
  "id": 42,
  "tracking_url": "https://track.tkani-05.ru/static/track.html?order=42",
  "qr_url": "/orders/42/qr"
}
```

Поле `tracking_url` нужно сохранить в 1С — это ссылка, которую печатает менеджер на чеке / отправляет покупателю.

**Ошибки:**

| HTTP | Причина | Действие в 1С |
|------|---------|---------------|
| 400 | Невалидные поля (например телефон не в формате +7...) | Показать пользователю текст из detail |
| 401 | Неверный X-API-Key | Уведомить администратора |
| 409 | Заказ с таким external_id уже существует в трекере | Не повторять отправку, считать успехом |
| 422 | Ошибка валидации схемы | Показать detail |
| 500 | Внутренняя ошибка сервера | Записать в журнал, повторить позже |

### 4.2. Обновление статуса заказа

`PATCH https://track.tkani-05.ru/api/v1/integration/orders/{external_id}/status`

Где `{external_id}` — номер заказа из 1С (например `/orders/ZK-2026-001234/status`).

**Тело запроса (JSON):**

```json
{
  "status": "in_production",
  "comment": "Передан в цех №2"
}
```

**Допустимые значения статуса:**

| Значение | Перевод |
|----------|---------|
| new | Принят |
| in_production | В производстве |
| ready | Готов |
| delivered | Доставлен |
| cancelled | Отменён |

**Успешный ответ (HTTP 200):** возвращает полную карточку заказа с историей этапов.

### 4.3. Получение статуса заказа

`GET https://track.tkani-05.ru/api/v1/integration/orders/{external_id}`

Возвращает текущий статус заказа и историю изменений. Может использоваться для синхронизации статусов «трекер → 1С», если потребуется в будущем.

## 5. Что должно быть сделано в 1С

### 5.1. Справочник «Установщики»

В справочнике установщиков 1С нужно добавить два реквизита:

- **ID в трекере** (число, необязательное) — соответствует полю `installer_id` в API трекера.
- **Контактный телефон** (строка) — должен храниться в формате +7XXXXXXXXXX. Совпадает с полем `phone` в карточке установщика трекера.

Заказчик готов прислать выгрузку всех установщиков с их ID из трекера в формате CSV / Excel — можно сразу заполнить реквизит «ID в трекере» массово.

### 5.2. Реквизиты заказа

В документ-заказ 1С добавить:

- **Номер в трекере** (число, заполняется ответом API)
- **Ссылка для покупателя** (строка, заполняется из `tracking_url`)
- **Дата последней отправки в трекер** (дата/время)
- **Текущий статус в трекере** (строка)

### 5.3. Действия

1. **При проведении / сохранении заказа** — отправить `POST /api/v1/integration/orders`, сохранить полученный `id` и `tracking_url` в реквизитах документа.
2. **При изменении статуса заказа в 1С** — отправить `PATCH /api/v1/integration/orders/{external_id}/status`. Сопоставить статусы 1С со статусами трекера согласно таблице в п. 4.2.
3. **Кнопка «Открыть карточку в трекере»** на форме документа — открывает `tracking_url` во внешнем браузере.
4. **Кнопка «Отправить SMS покупателю»** (опционально) — формирует текст со ссылкой `tracking_url`.

### 5.4. Обработка ошибок

- Все ошибки HTTP-запроса (таймаут, сетевая ошибка, ответ ≥ 500) — записывать в журнал регистрации с уровнем «Ошибка». Не блокировать работу пользователя.
- При HTTP 409 (заказ уже есть) — считать успехом, обновить локальные реквизиты через GET.
- При HTTP 400 / 422 — показать пользователю сообщение из `detail`.

### 5.5. Рекомендации по реализации

- Использовать встроенный объект 1С `HTTPСоединение` с включённым SSL (HTTPS).
- API-ключ хранить в константе или регистре настроек, не в коде.
- Сериализация JSON: `ЗаписьJSON` / `ПрочитатьJSON` (встроенные в 1С).
- Таймаут запроса — 30 секунд.

## 6. Тестирование

Перед продакшеном программист должен прислать заказчику:

1. Лог трёх тестовых запросов: создание заказа + смена статуса + получение статуса.
2. Скриншот формы документа 1С с заполненными реквизитами «Номер в трекере» и «Ссылка для покупателя».
3. Демонстрацию работы кнопки «Открыть карточку в трекере».

Тестовые установщики уже заведены в трекере, ID и телефоны можно получить у заказчика.
